解决 java.lang.OutOfMemoryError: Metaspace

May 22, 2024 阅读时间：1 分钟 | 🍉 476 字
JVM

问题描述 链接到标题

近日发现有一个微服务在运行时出现了 java.lang.OutOfMemoryError: Metaspace 的错误。这个错误是由于 Metaspace 区域内存不足导致的。Metaspace 是 Java 8 之后取代了永久代的区域，用于存放类的元数据信息。Metaspace 是在本地内存中分配的，不受 Java 堆大小的限制，但是受本地内存大小的限制。 当 Metaspace 区域内存不足时，会抛出 java.lang.OutOfMemoryError: Metaspace 错误。Metaspace 在 Java 虚拟机内存中的位置如图1所示：

临时解决方案 链接到标题

在出现 java.lang.OutOfMemoryError: Metaspace 错误时，可以通过增加 Metaspace 区域的内存大小来解决。在启动 Java 应用时，可以通过 -XX:MaxMetaspaceSize 参数来设置 Metaspace 区域的最大内存大小。例如，设置 Metaspace 区域的最大内存大小为 512M，可以通过如下命令来启动 Java 应用：

java -XX:MaxMetaspaceSize=512m -jar your-application.jar

长期解决方案 链接到标题

  • 排除微服务中加载的非必要的类和 JAR 包以及是否有代码动态加载不必要的类，从而减少 Metaspace 区域的内存占用。
  • 计算或统计实际 Matespace 所需的大小，设置合理的 Metaspace 区域内存大小，避免过大或过小。
