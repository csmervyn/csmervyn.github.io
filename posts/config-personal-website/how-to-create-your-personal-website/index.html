  • 使用哪个内容托管平台托管自己的静态网友

1.1.3 配置 Hugo 的主题 链接到标题

选择主题：Hugo 主题

将主题应用与自己的博客，在我们的示例中我们使用 hugo-coder 这个主题。

git submodule add https://github.com/luizdepra/hugo-coder.git themes/hugo-coder

在配置文件中表明自己的主题。

echo "theme = 'hugo-coder'" >> config.toml

1.1.4 创建一个空的博客 链接到标题

hugo new site csmervyn-blog

网站的初始结构：

.
├── archetypes
├── assets      # 静态资源存放处
├── config.toml # 配置文件入口
├── content     # 博文 markdown 文件
├── data
├── layouts
├── public      # 发布的静态站点存放处
├── resources
├── static
└── themes      # 主题的存放处
    └── hugo-coder

在当前目录初始化一个空的 git 仓库。

cd csmervyn-blog
git init

1.2 创建第一篇博文 链接到标题

hugo new posts/config-personal-website/how-to-create-your-personal-website.md

创建的文章位于根目录的content/posts/config-personal-website/ 下，博文的文件名为 how-to-create-your-personal-website.md。

我们用编辑器打开该文件，该文件的默认内容为：

---
title: "How to Create Your Personal Website"
date: 2023-05-04T13:31:11+08:00
draft: true
---

现在我们使用 markdown 的语法写自己的博客了。

1.2.1 预览博文 链接到标题

hugo server --buildDrafts
hugo server -D

运行服务后，我们可以通过浏览器 http://localhost:1313 地址访问我们的本地预览网页。

1.2.2 发布博文 链接到标题

在发布博文之前，需要删除文首的 draft: true。

hugo

当你发布站点时，Hugo 会在项目根目录的 public 目录中创建整个静态站点。这包括 HTML 文件和资产，例如图像、CSS 文件和 JavaScript 文件。

2 将博客内容放置于 Github pages 链接到标题

使用 GitHub action 将博客源码仓库的 public 目录中静态站点推送至个人的 Github pages。 

# Sample workflow for building and deploying a Hugo site to GitHub Pages
name: Deploy Hugo site to Pages

on:
  push:
    branches:
      - master  # 这里的意思是当 main分支发生push的时候，运行下面的jobs，这里先改为github-actions
  workflow_dispatch:

jobs:
  deploy: # 任务名自取
    runs-on: ubuntu-latest	# 在什么环境运行任务
    steps:
      - uses: actions/checkout@v3	# 引用actions/checkout这个action，与所在的github仓库同名
        with:
          submodules: true  # Fetch Hugo themes (true OR recursive) 获取submodule主题
          fetch-depth: 0    # Fetch all history for .GitInfo and .Lastmod

      - name: Setup Hugo	# 步骤名自取
        uses: peaceiris/actions-hugo@v2	# hugo官方提供的action，用于在任务环境中获取hugo
        with:
          hugo-version: 'latest'	# 获取最新版本的hugo
          extended: true

      - name: Build
        run: hugo --minify	# 使用hugo构建静态网页

      - name: Deploy
        uses: peaceiris/actions-gh-pages@v3	# 一个自动发布github pages的action
        with:
          external_repository: csmervyn/csmervyn.github.io	# 发布到哪个repo
          personal_token: ${{ secrets.ACTION_ACCESS_TOKEN }}	# 发布到其他repo需要提供上面生成的personal access token
          publish_dir: ./public	# 注意这里指的是要发布哪个文件夹的内容，而不是指发布到目的仓库的什么位置，因为hugo默认生成静态网页到public文件夹，所以这里发布public文件夹里的内容
          publish_branch: master	# 发布到哪个branch

上门的 github action 将博客源码中的 public 文件夹中的内容，推送到 csmervyn/csmervyn.github.io 的 master 分支。

3 安装评论插件 链接到标题

Github giscus app

  • giscus 主页
  • 选择安装 giscus 的 github account
  • 安装 giscus
  • confirm access
  • 完成安装 giscus

获取 giscus 需要的参数： coder 主题支持 Giscus插件，但需要按照 coder 的文档需要在 config.toml 中配置下面的内容。

  repo = "your-github-username/your-repo"
  repoID = "your-repo-id"
  category = "category"
  categoryID = "category-id"
  mapping = "pathname"
  term = ""
  strict = ""
  reactionsEnabled = "1"
  emitMetadata = "0"
  inputPosition = "bottom"
  theme = "light"
  lang = "zh-CN"
  loading = ""

其中这些repo、repoID、category 和 categoryID 等可以通过giscus官网获得。在其官网中填写自己的 repo 名称，选择语言等，会自动生成其参数。
