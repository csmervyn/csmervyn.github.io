图1-1 下载 ARM 架构的 Memory Analyzer

troubleshooting 链接到标题

问题1: The JVM shared library … does not contain the JNI_CreateJavaVM symbol. 链接到标题

错误信息图1-2所示。 details:

The JVM shared library "/Library/Java/JavaVirtualMachines/temurin-20.jdk/Contents/Home/bin/../lib/server/libjvm.dylib"

does not contain the JNI_CreateJavaVM symbol.

图1-2 The JVM shared library does not contain the JNI_CreateJavaVM symbol

解决方案 链接到标题

如图1-3 展示，Memory Analyzer 的 info.plist 文件和 MemoryAnalyzer.ini 文件的位置。

图1-3 info.plist 文件和 MemoryAnalyzer.ini 文件的位置

  • 修改 info.plist 文件，在 Eclipse 的 key 中添加 -vm 参数，指定 JVM 的路径。
<string>vm</string>
<string>/opt/homebrew/Cellar/openjdk@17/17.0.9/libexec/openjdk.jdk/Contents/Home/bin/java</string>
  • 修改 MemoryAnalyzer.ini 文件，在文件末尾添加 -vm 参数，指定 JVM 的路径。
-vm
/opt/homebrew/Cellar/openjdk@17/17.0.9/libexec/openjdk.jdk/Contents/Home/bin/java

问题2: 读取hprof文件时，The HPROF parser encountered a violation of the HPROF specification that it could not safely handle. 链接到标题

如图 1-4 所示，读取hprof文件时，遇到错误。 details:

The HPROF parser encountered a violation of the HPROF specification that it could not safely handle. This could be due to file truncation or a bug in the JVM. Please consider filing a bug at eclipse.org. To continue parsing the dump anyway, you can use -DhprofStrictnessWarning=true or set the strictness mode under Preferences > HPROF Parser > Parser Strictness. See the inner exception for details.

The HPROF parser encountered a violation of the HPROF specification that it could not safely handle. This could be due to file truncation or a bug in the JVM. Please consider filing a bug at eclipse.org. To continue parsing the dump anyway, you can use -DhprofStrictnessWarning=true or set the strictness mode under Preferences > HPROF Parser > Parser Strictness. See the inner exception for details.

(Possibly) Invalid HPROF file: Expected to read another 1,048,566 bytes, but only 597,351 bytes are available for heap dump record 0x1c at 0x5fceb290.

(Possibly) Invalid HPROF file: Expected to read another 1,048,566 bytes, but only 597,351 bytes are available for heap dump record 0x1c at 0x5fceb290.

图1-4 读取hprof文件时遇到的错误

解决方案 链接到标题

修改 Memory Analyzer 的 MemoryAnalyzer.ini 文件，在文件末尾添加 -DhprofStrictnessWarning=true 参数。

-DhprofStrictnessWarning=true
