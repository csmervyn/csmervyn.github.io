How to Set Username and Email in Git

December 26, 2023 阅读时间：1 分钟 | 🍉 138 字
Mervyn
Git

Git 中的 user.name 和 user.email 的 scope 的范围可以为单个 repository 的或全局的。利用这点上下文，我们可以很好解决多个 repository 中的 commit 需要不同 Author 的问题。下面首先分别阐述如何设置 scope 为单个 repository 和如何设置全局的的user.name 和 user.email。

Git 中的 user.name 和 user.email 的作用范围可分为单个 repository 和全局两种。这一特性使得我们能够有效解决在多个 repository 中进行 commit 时需要使用不同 Author 信息的问题。以下将分别详细阐述如何将 user.name 和 user.email 的作用范围设置为单个 repository 和全局。

将 user.name 和 user.email 的作用范围设置为单个 repository 链接到标题

  1. 进入到 repository 的根目录

  2. 设置 Git username 和 email

    git config user.name csmervyn
git config user.email csmervyn@gmail.com

  3. 确认设置是否生效

    git config user.name
git config user.email

将 user.name 和 user.email 的作用范围设置为全局 链接到标题

  1. 可以在任意目录

  2. 设置 Git username 和 email

    git config --global user.name csmervyn
git config --global user.email csmervyn@gmail.com

  3. 确认设置是否生效

    git config --global user.name
git config --global user.email

最佳实践 链接到标题

原则也很简单。如果多数 repository 需要的用户名和邮箱，通常设置为全局的 user.name 和 user.email；个别 repository 需要的特殊用户名和邮箱，每个 repository 单独设置。例如在公司电脑，全局的 user.name 和 user.email 设置成工号和公司域名的邮箱；个人私有的 repository 单独设置成自己的名称和私人邮箱。
